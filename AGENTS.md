# AGENTS.md

This repository is an academic research template built around R, Quarto, `renv`, and `resultcheck`.

## Environment
- Use R 4.6.0.
- Restore dependencies with `renv::restore()`.
- Quarto is required to render the manuscript and slides.
- `tinytex` is required for LaTeX-based rendering.

## Repository layout
- `code/`: numbered analysis scripts that should be run in order
- `data_raw/`: immutable source data
- `data_processed/`: cleaned data generated by the scripts
- `outputs/`: generated analysis outputs
- `manuscript/` and `slides/`: Quarto sources for publication-ready outputs
- `tests/testthat/`: regression tests for the example workflow
- `tests/_resultcheck_snaps/`: `resultcheck` snapshots used to verify stable results

## Expected workflow
1. Restore the project library with `renv::restore()`.
2. Run `code/1_data_cleaning.R`.
3. Run `code/2_descriptive.R`.
4. Run `code/3_main_analysis.R`.
5. Run `testthat::test_dir("tests/testthat")`.
6. Render `manuscript/manuscript.qmd` or `slides/slides.qmd` as needed.

## Testing and reproducibility
- The example scripts use `resultcheck::snapshot()` to capture stable intermediate outputs.
- The regression test entry point is `Rscript -e "testthat::test_dir('tests/testthat')"`.
- Keep documentation aligned with `renv.lock` and `.github/workflows/run-tests.yml` when the R version or test workflow changes.
